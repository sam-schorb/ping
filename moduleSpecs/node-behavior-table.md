# Node Behavior Table

This table assumes:

- Value range is **1–8**.
- Each node has **one parameter**, default **1** (unless noted as no‑param).
- **Default port layout (single‑input nodes):**
  - **left‑top = main input**
  - **left‑bottom = control input** (if the node accepts control)
  - **right‑top = main output**
- **Control pulses only update node control state/params** and **do not pass through**.
- If **control and main input** arrive in the same tick, **control updates first**, then main input is processed with the new control state.
- Results are **clamped to 1–8**.
- Pulses entering the **output side** are ignored (one‑directional).
- **All pulses arriving in the same tick are processed independently** (polyphonic).
- **Pulse speed and params pass through unchanged** unless a node explicitly overwrites them.
- **Port order indices are 0‑based in code**; this table uses a human‑readable order.
- **Port naming:** _left‑top/left‑bottom_ are on the **left side**. _top‑left/top‑right_ are on the **top edge** (used in mux ordering).
- The current canonical graph DSL surface is described in `./09-DSL.md`.
- This table is behavior-first, not syntax-first; outdated Hydra-style DSL aliases such as `p()` and `kN()` are not part of the current canonical plan.
- **Group nodes are dynamic** (defined in project group library) and are not listed in the main table; their current DSL shape is documented in `./09-DSL.md`.

> **Multi‑I/O nodes** (`mux`, `demux`, `switch`) use the explicit port maps below.

| ------------------------- | --------------- | ------------------------------------------------------------------------------------------- | ----------------------------------- | -------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |

## DSL Shape Summary

The current implemented node set falls into a small number of user-facing DSL shape families.
The canonical syntax lives in `./09-DSL.md`; this appendix is only a concise bridge from node behavior to DSL shape.

| Family | Node types | Canonical DSL shape | Notes |
| --- | --- | --- | --- |
| Single-stream chain nodes | `pulse`, `add`, `sub`, `set`, `speed`, `pitch`, `decay`, `crush`, `hpf`, `lpf`, `block`, `every`, `drop`, `random`, `counter`, `gtp`, `ltp`, `gtep`, `ltep`, `match` | `source.node(param){control}` or `name = node(param){control}` plus a separate signal wire | Param clause optional. Control clause optional. The current built-in visible nodes expose at most one live control clause. |
| Sink terminal | `out` | `expr.out()` | `output` is only an alias, not the canonical DSL spelling. |
| Multi-output indexed | `mux`, `switch` | `name = source.mux()` or `name = source.switch(param){control}` then `name[index]` | Indexed outputs are 0-based. |
| Multi-input indexed | `demux` | `name = demux()` then `source.name[index]` | Indexed inputs are 0-based. |
| Dynamic expanded form | `group` | inline-expanded into ordinary visible nodes | Raw `groupRef` is not surfaced canonically. |

## DSL Index Mapping For Multi-Port Nodes

This table turns the human-facing physical port order above into the exact 0-based indices used by the canonical DSL.

| DSL index | `mux` output | `switch` output | `demux` input |
| --- | --- | --- | --- |
| `0` | top-left | top-left | top-right |
| `1` | top-right | top-right | top-left |
| `2` | right-top | right-top | left-top |
| `3` | right-bottom | right-bottom | left-bottom |
| `4` | bottom-right | bottom-right | bottom-left |
| `5` | bottom-left | bottom-left | bottom-right |

These index mappings should stay in sync with:

- the current port-ordering note above
- `packages/core/nodes/archetypes.js`
- `./09-DSL.md`
